SaaS Control de Alícuotas
Plataforma SaaS multi-cliente para conjuntos habitacionales

Control inteligente de pagos y accesos peatonales/vehiculares

Solución web profesional para fabricantes y administradores de conjuntos habitacionales: gestión de alícuotas, copropietarios, arrendatarios, unidades habitacionales, reportes y automatización del acceso según estado de pago. Integra conceptualmente con Soyal y Tuya Smart.

1. Descripción general del sistema

La plataforma opera como SaaS con aislamiento por cliente (tenant), permitiendo que el fabricante administre múltiples conjuntos habitacionales desde un panel central. Cada cliente dispone de su propio entorno para gestionar pagos de alícuotas y operaciones administrativas, mientras que los copropietarios visualizan información de pagos, deudas y notificaciones.

2. Arquitectura del sistema

Frontend

SPA web responsiva con paneles por rol, dashboard, reportes y centro de notificaciones.

Backend API

Servicios REST para autenticación, facturación, gestión de usuarios, reglas de acceso e integraciones externas.

Base de datos

Modelo relacional multi-tenant con separación lógica por cliente y auditoría de operaciones.

Integraciones

Conectores API para Soyal y Tuya Smart con cola de eventos, reintentos y trazabilidad.

Motor de reglas

Evalúa estado de cartera y aplica bloqueo/desbloqueo automático de accesos.

Observabilidad

Logs estructurados, métricas y alertas para uso del fabricante y soporte operativo.

3. Módulos principales

  • Gestión de clientes SaaS (alta, activación, suspensión y licencias).
  • Usuarios y roles con permisos granulares.
  • Registro de copropietarios, arrendatarios y unidades habitacionales.
  • Facturación y control de pagos de alícuotas.
  • Reportes de pagos, mora y estado de cartera.
  • Notificaciones automatizadas (email, app, panel).
  • Integración con sistemas de control de acceso.
  • Motor de reglas para bloqueo/desbloqueo automático.

4. Tipos de usuarios

Fabricante (Super Admin)

Crea cuentas de clientes, gestiona licencias, habilita/deshabilita acceso y monitorea la plataforma completa.

Administrador del cliente

Gestiona datos del conjunto: unidades, personas, pagos, reportes y políticas administrativas.

Copropietario (solo visualización)

Consulta pagos realizados, deudas pendientes, notificaciones e información administrativa sin permisos de edición.

5. Flujo del sistema

  1. El fabricante registra un nuevo cliente y configura su tenant.
  2. El administrador del cliente crea usuarios, unidades y vínculos con residentes.
  3. Se generan alícuotas y se actualiza el estado de cuenta por unidad.
  4. El motor de reglas evalúa mora y determina estado de acceso.
  5. Se envía evento a Soyal/Tuya para bloquear o habilitar acceso.
  6. Los copropietarios visualizan su estado financiero y notificaciones en tiempo real.

6. Estructura de base de datos (conceptual)

  • tenants: información del cliente SaaS.
  • users: usuarios y credenciales.
  • roles, permissions: control de acceso.
  • units: viviendas/unidades habitacionales.
  • residents: copropietarios/arrendatarios.
  • fees: alícuotas y periodos.
  • payments: pagos registrados y comprobantes.
  • debts: saldos pendientes y mora.
  • access_rules, access_events: decisiones y eventos de acceso.
  • audit_logs, notifications: trazabilidad y comunicación.

8. Lógica de negocio para control de pagos y accesos

Regla de estado financiero

Si existe deuda vencida > umbral configurado, se marca unidad en estado de mora.

Acción de bloqueo

Estado en mora genera evento de bloqueo de acceso peatonal y/o vehicular.

Rehabilitación automática

Al confirmar pago y conciliación, se emite evento de desbloqueo automáticamente.

Pagos al día → Acceso activo Saldo próximo a vencer → Notificación preventiva Mora vencida → Bloqueo automático

9. Integración conceptual con Soyal y Tuya Smart

Soyal

  • Mapeo de unidades/usuarios a identificadores del dispositivo.
  • Comandos de alta/baja de autorización y estado de acceso.
  • Reintentos automáticos y bitácora de resultados por evento.

Tuya Smart

  • Integración mediante API Cloud para dispositivos compatibles.
  • Sincronización de permisos y acciones remotas por regla de negocio.
  • Trazabilidad de cambios para auditoría y soporte.

10. Tecnologías recomendadas

Frontend: React + TypeScript + Vite UI: Tailwind CSS / Material UI Backend: Node.js (NestJS) o .NET DB: PostgreSQL Cache/colas: Redis + RabbitMQ Auth: JWT + Refresh Tokens Infra: Docker + Kubernetes CI/CD: GitHub Actions Observabilidad: OpenTelemetry + Grafana

11. Seguridad

  • Autenticación robusta y hashing de contraseñas (Argon2/Bcrypt).
  • Autorización por roles y permisos por tenant.
  • Cifrado TLS en tránsito y cifrado selectivo en reposo.
  • Rate limiting y protección contra abuso de APIs.
  • Registro de auditoría para acciones críticas.
  • Políticas de sesión segura y revocación de tokens.
  • Validación estricta de entradas y hardening del backend.
  • Backups automáticos y estrategia de recuperación.
